Prompt Adaptation Guide

How to Use This Vault

Layer 1: Tune the Tone

The way you speak to your customers shapes how your agents should write. If your brand is formal and technical, update the tone instruction in every prompt. If you're casual and direct, say so. One line at the top of any prompt — "Write in a [TONE] voice, for example: [EXAMPLE]" — changes the output dramatically. Don't leave tone to chance.

Layer 2: Add Context

The more an agent knows about your specific situation, the better it performs. Before running any prompt, add a context block: your business type, the customer segment you're addressing, relevant history, and any constraints. Agents don't make assumptions well. You fill the gaps, they execute cleanly. A 3-sentence context block at the top of any prompt doubles output quality.

Layer 3: Specify the Output Format

If you need a table, say so. If you need a Google Doc-ready format, say so. If you need three options instead of one, ask for three. The default output is a draft. Specify the format you actually need and skip the reformatting step. This one change saves 10 minutes per prompt, minimum.

Chaining Prompts Inside OpenClaw

Run prompt one and save the output as a variable or file. Reference that output explicitly in the next prompt. For example: "Using the subject lines generated in the previous step, write the full email body for the top-performing option." Each agent in your org can own a section of the chain. Marketing generates subject lines. Copywriting writes the body. Support drafts the follow-up. One vault, infinite workflows.

Saving and Reusing Prompts in OpenClaw

Store your best prompts as SKILL.md files in your agent's workspace, or keep a prompts directory your agents can reference by file path. Name them clearly. A prompt called sales-cold-email-v2.md is easier to call than remembering the full text. Version your prompts. When you improve one, save a new version — don't overwrite what worked. Build your own prompt library and your agents get smarter every week.
